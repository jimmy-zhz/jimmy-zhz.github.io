# 第一部分：关联规则（Association Rules）

## 1.1 问题定义

**关联规则挖掘**是从大规模数据集中发现变量之间有趣关联关系的任务。最经典的案例是"啤酒与尿布"——沃尔玛发现购买啤酒的顾客往往同时购买尿布。

### 形式化定义

设 $I = \{i_1, i_2, ..., i_m\}$ 是所有项（item）的集合，称为**项集**（Itemset）。设 $D$ 是事务数据库（Transaction Database），每个事务 $T$ 是 $I$ 的子集，即 $T \subseteq I$。

**关联规则**是形如 $X \Rightarrow Y$ 的蕴含式，其中：
- $X \subset I$，$Y \subset I$
- $X \cap Y = \emptyset$（项集不相交）

### 核心度量指标

| 指标 | 公式 | 含义 |
|------|------|------|
| **支持度（Support）** | $supp(X) = \frac{|{t \in D: X \subseteq t}|}{|D|}$ | 项集在数据库中出现的频率 |
| **置信度（Confidence）** | $conf(X \Rightarrow Y) = \frac{supp(X \cup Y)}{supp(X)}$ | 包含 X 的事务中同时包含 Y 的比例 |
| **提升度（Lift）** | $lift(X \Rightarrow Y) = \frac{supp(X \cup Y)}{supp(X) \times supp(Y)}$ | 关联强度相对于随机期望的提升 |

**解读**：
- 支持度 > 阈值（如 1%）→ 项集是"频繁"的
- 置信度 > 阈值（如 50%）→ 规则有"强"关联
- Lift > 1 → 正相关；Lift < 1 → 负相关；Lift = 1 → 独立

## 1.2 Apriori 算法

### 算法思想

**先验原理（Apriori Principle）**：如果一个项集是频繁的，那么它的所有子集也一定是频繁的。反之，如果一个项集是非频繁的，那么它的所有超集也都是非频繁的。

### 算法流程

```
输入：事务数据库 D，最小支持度阈值 min_sup
输出：所有频繁项集

1. 扫描数据库，统计每个 1-项集的支持度
2. 剪枝：保留支持度 ≥ min_sup 的 1-项集，形成 L₁
3. 迭代直到没有新的频繁项集产生：
   a. 由 Lₖ₋₁ 连接生成候选 k-项集 Cₖ
   b. 扫描数据库，计算 Cₖ 中每个候选的支持度
   c. 剪枝：保留支持度 ≥ min_sup 的项集，形成 Lₖ
4. 返回所有 Lₖ 的并集
```

### 示例说明

假设事务数据库 D 包含 5 条交易：

| TID | 购买商品 |
|-----|---------|
| T1 | {面包, 牛奶} |
| T2 | {面包, 尿布, 啤酒, 鸡蛋} |
| T3 | {牛奶, 尿布, 啤酒, 可乐} |
| T4 | {面包, 牛奶, 尿布, 啤酒} |
| T5 | {面包, 牛奶, 可乐} |

最小支持度设为 40%（即出现 ≥ 2 次）：

1. **1-项集支持度**：
   - {面包}: 4/5 = 80% ✓
   - {牛奶}: 4/5 = 80% ✓
   - {尿布}: 3/5 = 60% ✓
   - {啤酒}: 3/5 = 60% ✓
   - {鸡蛋}: 1/5 = 20% ✗
   - {可乐}: 2/5 = 40% ✓

2. **生成 2-项集**（剪枝后）：
   - {面包, 牛奶}: 3/5 = 60% ✓
   - {面包, 尿布}: 2/5 = 40% ✓
   - {面包, 啤酒}: 2/5 = 40% ✓
   - {牛奶, 尿布}: 2/5 = 40% ✓
   - ...

3. **生成 3-项集**：
   - {面包, 尿布, 啤酒}: 2/5 = 40% ✓

### 算法复杂度

- **时间复杂度**：$O(m \times 2^m)$，其中 m 为项数
- **空间复杂度**：需要存储大量候选集
- **缺点**：需要多次扫描数据库，产生大量候选集

## 1.3 FP-Growth 算法

### 算法思想

FP-Growth（ Frequent Pattern Growth）通过构建 **FP-Tree（频繁模式树）** 压缩数据库，避免生成候选集。

### 核心概念

1. **FP-Tree**：一种前缀树结构，相同前缀共享节点
2. **项头表（Header Table）**：记录每个项在 FP-Tree 中的所有节点位置

### 算法流程

```
1. 扫描数据库，统计每个项的支持度，降序排列
2. 构建 FP-Tree：
   - 按支持度排序事务
   - 插入前缀树，相同前缀共享节点
3. 对每个项，构造条件模式基（Conditional Pattern Base）
4. 递归挖掘条件 FP-Tree，生成频繁项集
```

### 优势对比

| 特性 | Apriori | FP-Growth |
|------|---------|-----------|
| 候选集 | 需生成大量候选 | 无候选集 |
| 扫描次数 | 多次扫描数据库 | 两次扫描 |
| 效率 | 较低 | 较高 |
| 内存 | 候选集占用大量内存 | FP-Tree 压缩存储 |

## 1.4 关联规则评估与优化

### 评估指标选择

| 场景 | 推荐指标 | 原因 |
|------|---------|------|
| 寻找普遍模式 | 支持度 | 筛选高频项集 |
| 预测性规则 | 置信度 | 衡量规则预测能力 |
| 去除虚假关联 | 提升度 | 排除随机共现 |

### 规则剪枝策略

1. **提升度剪枝**：移除 Lift ≤ 1 的规则
2. **冗余规则移除**：如果 $X \Rightarrow Y$ 的置信度等于 $X' \Rightarrow Y$（$X' \subset X$），则 $X' \Rightarrow Y$ 是冗余的
3. **多层关联**：可设置项的层次结构（如：食品 → 水果 → 苹果）

# 第二部分：时间序列分析（Time Series Analysis）

## 2.1 问题定义

**时间序列**是按时间顺序排列的观测值序列 $\{x_1, x_2, ..., x_t, ...\}$，其中 $t$ 表示时间索引。

### 时间序列的组成成分

一个时间序列通常可分解为四个成分：

$$X_t = T_t + S_t + C_t + R_t$$

| 成分 | 描述 | 示例 |
|------|------|------|
| **趋势（Trend）$T_t$** | 长期变化方向 | 人口增长、GDP 上升 |
| **季节性（Seasonality）$S_t$** | 周期性波动 | 冰淇淋销量夏季高峰 |
| **循环（Cycle）$C_t$** | 不规则长周期波动 | 经济周期（8-10年） |
| **残差（Residual）$R_t$** | 随机噪声 | 随机波动 |

### 平稳性（Stationarity）

**平稳时间序列**的核心特征：
- **均值常数**：$E[x_t] = \mu$，不随时间变化
- **方差常数**：$Var[x_t] = \sigma^2$，不随时间变化
- **自协方差仅依赖于滞后**：$Cov(x_t, x_{t-k}) = \gamma_k$

**为什么重要**：许多时间序列模型（如 ARMA）要求序列平稳，否则会产生"伪回归"。

---

## 2.2 时间序列数据处理

### 数据划分策略

| 方法 | 描述 | 适用场景 |
|------|------|---------|
| **固定划分** | 按固定时间点划分训练/测试集 | 预测固定未来时间段 |
| **滑动窗口** | 窗口滑动生成重叠的输入-输出对 | 模型训练，需保留时间依赖 |
| **时间交叉验证** | 按时间顺序逐步扩大训练集 | 评估模型稳定性 |

### 平稳化处理

#### 1. 差分（Differencing）

**一阶差分**：$\nabla x_t = x_t - x_{t-1}$

**季节性差分**：$\nabla_s x_t = x_t - x_{t-s}$（s 为季节周期）

**作用**：
- 消除趋势使均值平稳
- 稳定方差
- 揭示序列的长期依赖结构

#### 2. 对数变换

$$x'_t = \log(x_t + c)$$

**作用**：稳定方差（适用于乘法季节性）

#### 3. Box-Cox 变换

$$x'_t = \begin{cases} \frac{(x_t + c)^\lambda - 1}{\lambda} & \lambda \neq 0 \\ \log(x_t + c) & \lambda = 0 \end{cases}$$

**作用**：自动寻找最优幂变换使数据接近正态分布

### 缺失值处理

| 方法 | 描述 | 适用场景 |
|------|------|---------|
| **前向填充** | 用前一个观测值填充 | 短期缺失 |
| **后向填充** | 用后一个观测值填充 | 未来信息已知时 |
| **线性插值** | 假设线性变化 | 趋势平稳区域 |
| **样条插值** | 假设平滑变化 | 波动较大区域 |

## 2.3 时间序列特征工程

### 滞后特征（Lag Features）

将历史观测值作为特征：

$$x_{t-k} \rightarrow \text{lag}_k$$

### 滚动统计特征

| 特征 | 公式 | 含义 |
|------|------|------|
| **滚动均值** | $\mu_t = \frac{1}{k}\sum_{i=t-k+1}^{t} x_i$ | 平滑趋势 |
| **滚动标准差** | $\sigma_t = \sqrt{\frac{1}{k}\sum_{i=t-k+1}^{t}(x_i - \mu_t)^2}$ | 波动性 |
| **滚动最大/最小** | $\max/min$ | 极值特征 |

### 时间分解特征

| 特征 | 描述 |
|------|------|
| **傅里叶变换** | 提取周期成分的频率和振幅 |
| **小波变换** | 多尺度时频分析 |
| **STL 分解** | 季节性趋势分解（Seasonal and Trend decomposition using Loess） |

## 2.4 经典时间序列模型

### 2.4.1 统计模型

#### 自回归模型 AR(p)

$$x_t = \phi_0 + \phi_1 x_{t-1} + \phi_2 x_{t-2} + ... + \phi_p x_{t-p} + \varepsilon_t$$

**思想**：当前值是过去 p 个观测值的线性组合

#### 移动平均模型 MA(q)

$$x_t = \mu + \varepsilon_t + \theta_1 \varepsilon_{t-1} + ... + \theta_q \varepsilon_{t-q}$$

**思想**：当前值是过去 q 个随机误差的线性组合

#### ARMA 模型

$$x_t = \phi_0 + \sum_{i=1}^{p} \phi_i x_{t-i} + \varepsilon_t + \sum_{j=1}^{q} \theta_j \varepsilon_{t-j}$$

**思想**：结合自回归和移动平均

#### ARIMA 模型

**Integrated（差分）**：对非平稳序列进行 d 阶差分后使用 ARMA

| 参数 | 含义 |
|------|------|
| p | AR 阶数（自回归项） |
| d | 差分阶数（使序列平稳） |
| q | MA 阶数（移动平均项） |

**SARIMA**：加入季节性成分 $P \times D \times Q \times S$

#### 指数平滑法

| 方法 | 公式 | 适用场景 |
|------|------|---------|
| **简单指数平滑** | $\hat{x}_{t+1} = \alpha x_t + (1-\alpha)\hat{x}_t$ | 无趋势无季节 |
| **Holt 线性趋势** | 叠加趋势项 | 有趋势无季节 |
| **Holt-Winters** | 叠加趋势+季节项 | 有趋势有季节 |

### 2.4.2 机器学习模型

| 模型 | 特点 | 适用场景 |
|------|------|---------|
| **随机森林** | 可处理非线性关系，需特征工程 | 中短期预测 |
| **SVR** | 核方法处理非线性 | 小样本预测 |
| **KNN** | 基于相似度预测 | 简单基线模型 |
| **GBDT** | 特征交互能力强 | 复杂模式 |

### 2.4.3 深度学习模型

#### RNN（循环神经网络）

**核心思想**：隐藏状态 $h_t$ 包含过去信息，递归传递

$$h_t = \tanh(W_{hh}h_{t-1} + W_{xh}x_t + b)$$

**问题**：梯度消失/爆炸，难以捕捉长程依赖

#### LSTM（长短期记忆网络）

**三门机制**：
- **遗忘门**：决定丢弃什么信息
- **输入门**：决定存储什么新信息
- **输出门**：决定输出什么信息

**优势**：解决长程依赖问题

#### GRU（门控循环单元）

**两门机制**：
- **更新门**：控制过去信息传递到未来的程度
- **重置门**：控制忽略过去信息的程度

**优势**：参数更少，训练更快

#### Transformer

**核心机制**：自注意力（Self-Attention）

$$Attention(Q, K, V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**优势**：
- 并行计算，训练效率高
- 捕捉任意距离的依赖关系
- 位置编码保留序列顺序信息

## 2.6 隐马尔可夫模型（HMM）

### 模型定义

**隐马尔可夫模型**是用于描述含有隐含状态的马尔可夫过程的统计模型。

**基本假设**：
1. **马尔可夫性**：当前状态仅依赖于前一状态
2. **观测独立性**：观测值仅依赖于当前状态

### 核心要素

| 要素 | 描述 |
|------|------|
| **状态集合** $Q$ | 隐含状态的有限集合 $\{q_1, ..., q_N\}$ |
| **观测集合** $V$ | 可观测输出的集合 $\{v_1, ..., v_M\}$ |
| **初始概率** $\pi$ | 初始状态概率分布 |
| **状态转移矩阵** $A$ | $a_{ij} = P(q_j | q_i)$ |
| **观测概率矩阵** $B$ | $b_j(k) = P(v_k | q_j)$ |

### 三个基本问题

| 问题 | 描述 | 算法 |
|------|------|------|
| **评估（Evaluation）** | 给定模型 $\lambda$，计算观测序列概率 $P(O|\lambda)$ | 前向算法 |
| **解码（Decoding）** | 给定模型 $\lambda$ 和观测序列 $O$，找出最可能的隐状态序列 | Viterbi 算法 |
| **学习（Learning）** | 给定观测序列 $O$，学习模型参数 $\lambda$ | Baum-Welch 算法（EM） |
