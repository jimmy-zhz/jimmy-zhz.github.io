# 数据准备（Data Preparation）

> 本手册是机器学习生命周期的第三阶段，参照 CRISP-DM 框架编写。

---

## 1. 数据准备的重要性

**数据准备**是机器学习项目中最耗时的阶段，通常占据整个项目 **60%-80%** 的时间。正如一句数据科学界的格言所说："数据和特征决定了机器学习的上限，而模型和算法只是逼近这个上限。"

即使拥有最先进的算法和最强大的计算资源，如果数据质量不佳或特征不合适，模型性能也难以达到预期。因此，数据准备阶段的工作质量直接决定了最终模型的效果。

---

## 2. 数据收集

### 2.1 确定数据来源

机器学习项目的数据可能来自多种来源：

| 数据来源 | 示例 | 特点 |
|---------|------|------|
| **内部数据库** | 业务数据库、数据仓库 | 质量较高，但可能分散 |
| **日志系统** | 用户行为日志、系统日志 | 数据量大，但需要解析 |
| **第三方数据** | 外部API、数据供应商 | 可补充信息，但需付费 |
| **公开数据集** | Kaggle、UCI、政府数据 | 可用于验证，但可能不匹配业务 |
| **问卷调查** | 用户反馈、市场调研 | 成本高，但可获取特定信息 |

### 2.2 数据获取方式

- **批量导出**：定期从数据库导出数据
- **实时流式获取**：通过 Kafka、Kinesis 等流式平台
- **API 调用**：通过 RESTful API 获取外部数据
- **网页爬取**：从网站抓取数据（需遵守法律和robots.txt）

### 2.3 数据存储

根据数据特性和使用场景选择合适的存储方案：

| 存储类型 | 适用场景 | 优点 | 缺点 |
|---------|---------|------|------|
| **关系型数据库（SQL）** | 结构化数据、事务处理 | 成熟稳定、查询灵活 | 扩展性有限 |
| **数据仓库** | 大规模分析 | 优化查询、性能强大 | 成本高 |
| **数据湖** | 多格式原始数据 | 灵活、低成本 | 需要治理 |
| **NoSQL** | 非结构化数据 | 高扩展性 | 查询能力有限 |
| **对象存储** | 文件、图片、视频 | 成本低、无限扩展 | 不适合查询 |

## 3. 数据清洗

### 3.1 缺失值处理

#### 3.1.1 缺失值机制

根据统计学分类，缺失值可分为三种机制：

| 机制 | 英文 | 定义 | 示例 |
|------|------|------|------|
| **完全随机缺失** | MCAR | 缺失与任何变量无关 | 问卷随机丢失 |
| **随机缺失** | MAR | 缺失与观测变量相关 | 高收入者不填收入 |
| **非随机缺失** | MNAR | 缺失与未观测值相关 | 重病患者不复诊 |

#### 3.1.2 缺失值处理策略

| 策略 | 方法 | 适用场景 |
|------|------|---------|
| **删除** | 删除缺失记录或特征 | MCAR、缺失比例低 |
| **均值/中位数填充** | 用统计量替代 | 数值型、MCAR |
| **众数填充** | 用最频繁值替代 | 分类型 |
| **插值法** | 基于相邻值插值 | 时间序列 |
| **KNN插补** | 用相似样本的值替代 | MAR |
| **多重插补（MICE）** | 多次插补取平均 | 复杂缺失模式 |
| **缺失标记** | 创建"是否缺失"特征 | 缺失有信息价值 |

### 3.2 异常值处理

#### 3.2.1 异常值检测方法

| 方法 | 原理 | 适用场景 |
|------|------|---------|
| **Z-Score** | $\|Z\| > 3$ 为异常 | 正态分布数据 |
| **IQR方法** | 超出 $Q1-1.5IQR$ 或 $Q3+1.5IQR$ | 任意分布 |
| **修改Z-Score** | 基于中位数的鲁棒方法 | 非正态分布 |
| **Isolation Forest** | 基于孤立度的检测 | 高维数据 |
| **LOF** | 基于局部密度的检测 | 多维异常 |

#### 3.2.2 异常值处理策略

| 策略 | 方法 | 适用场景 |
|------|------|---------|
| **删除** | 直接移除异常记录 | 明确的数据错误 |
| **截断** | 将值限制在阈值范围内 | 保留样本量 |
| **变换** | 对数、Box-Cox变换 | 重尾分布 |
| **分箱** | 将连续变量离散化 | 减少异常影响 |
| **标记** | 创建异常标记特征 | 异常值有信息价值 |
| **单独建模** | 为异常值单独建模 | 异常来自不同总体 |

### 3.3 重复数据处理

| 问题 | 处理策略 |
|------|---------|
| **完全重复** | 删除重复记录，保留一条 |
| **部分重复** | 按关键字段去重，保留最新/最完整 |
| **重复标记** | 创建重复计数特征，保留信息 |

### 3.4 数据一致性处理

| 问题类型 | 处理方法 |
|---------|---------|
| **格式不一致** | 统一日期格式、字符串大小写 |
| **矛盾数据** | 业务规则验证和修正 |
| **时间逻辑错误** | 校验时间先后顺序 |
| **参照完整性** | 校验外键关系 |

## 4. 数据转换

### 4.1 数值缩放

| 方法 | 公式 | 特点 |
|------|------|------|
| **标准化（Z-Score）** | $z = \frac{x - \mu}{\sigma}$ | 均值为0，标准差为1 |
| **归一化（Min-Max）** | $x' = \frac{x - x_{min}}{x_{max} - x_{min}}$ | 映射到[0,1] |
| **对数变换** | $x' = \log(x + c)$ | 压缩右偏分布 |
| **Robust Scaler** | 基于IQR的缩放 | 对异常值鲁棒 |

### 4.2 分类变量编码

| 方法 | 描述 | 适用场景 |
|------|------|---------|
| **标签编码** | 用整数替换类别 | 树模型、高基数 |
| **独热编码** | 每个类别创建二元列 | 低基数、无序类别 |
| **目标编码** | 用目标均值替换类别 | 高基数、避免维度爆炸 |
| **有序编码** | 利用类别顺序 | 序数特征 |
| **频率编码** | 用类别频率替换 | 捕捉类别稀有性 |
| **嵌入编码** | 学习低维向量表示 | 深度学习场景 |

### 4.3 连续变量离散化

| 方法 | 描述 | 优点 |
|------|------|------|
| **等宽分箱** | 将范围等分为N份 | 简单直观 |
| **等频分箱** | 每箱包含相等数量的样本 | 适应数据分布 |
| **决策树分箱** | 用决策树确定最优切分点 | 最优分割 |
| **聚类分箱** | 用聚类算法确定分箱 | 捕捉非线性 |

### 4.4 数据变换

| 变换方法 | 目的 | 适用场景 |
|---------|------|---------|
| **对数变换** | 压缩右偏分布 | 金融数据、收入 |
| **Box-Cox变换** | 使数据接近正态分布 | 回归分析 |
| **平方根变换** | 稳定方差 | 计数数据 |
| **倒数变换** | 处理双曲线关系 | 特定场景 |

## 5. 数据划分

### 5.1 划分策略

| 划分方式 | 比例 | 适用场景 |
|---------|------|---------|
| **训练/测试** | 70:30 或 80:20 | 小数据集 |
| **训练/验证/测试** | 60:20:20 | 需要调参 |
| **交叉验证** | K-Fold | 数据有限 |

### 5.2 划分原则

**关键原则**：测试集必须与训练集完全隔离，确保评估的客观性。

| 原则 | 说明 |
|------|------|
| **代表性** | 各子集应反映整体数据的分布 |
| **独立性** | 样本在划分前应相互独立 |
| **时间序列划分** | 时序数据应按时间顺序划分 |
| **分层抽样** | 分类问题应保持类别比例 |

### 5.3 数据不平衡处理

当数据不平衡时（类别比例超过10:1），需要特殊处理：

| 方法 | 原理 | 适用场景 |
|------|------|---------|
| **过采样** | 复制少数类样本 | 样本总量足够 |
| **欠采样** | 减少多数类样本 | 样本总量充足 |
| **SMOTE** | 合成少数类样本 | 中度不平衡 |
| **类权重** | 调整损失函数权重 | 所有场景 |
| **阈值调整** | 调整分类阈值 | 二分类 |

## 6. 数据质量验证

### 6.1 数据质量维度

| 维度 | 定义 | 评估方法 |
|------|------|---------|
| **完整性** | 数据的完整程度 | 缺失值比例 |
| **准确性** | 数据与真实值的符合程度 | 与可靠来源对比 |
| **一致性** | 数据内部无矛盾 | 业务规则验证 |
| **时效性** | 数据是否最新 | 时间戳分析 |
| **唯一性** | 无重复记录 | 重复检测 |
| **有效性** | 数据符合定义的格式和范围 | 范围检查 |

### 6.2 数据质量报告

在数据准备完成后，应生成数据质量报告，包括：

- 原始数据的基本信息（行数、列数、存储大小）
- 数据清洗的详细记录（处理了哪些问题、如何处理）
- 数据转换的详细记录（应用了哪些转换）
- 数据划分的详细记录（各子集的大小和比例）
- 数据质量的最终评估

---

## 7. 数据准备检查清单

在进入建模阶段之前，确认以下事项：

- [ ] 数据来源已确定并可访问
- [ ] 缺失值已处理，记录处理方法
- [ ] 异常值已检测和处理
- [ ] 重复记录已处理
- [ ] 数据一致性已验证
- [ ] 数值特征已适当缩放
- [ ] 分类特征已适当编码
- [ ] 连续变量已适当离散化（如需要）
- [ ] 数据已按正确比例划分
- [ ] 训练集、验证集、测试集相互独立
- [ ] 数据质量已验证
- [ ] 数据准备报告已生成

---

## 8. 产出文档

### 8.1 清洗后的数据集

可直接用于建模的数据集，包括：
- 训练集
- 验证集
- 测试集

### 8.2 数据准备报告

详细记录数据准备过程中的所有决策和操作，包括：
- 数据来源和获取方式
- 数据清洗的方法和理由
- 数据转换的方法和参数
- 数据划分的比例和方式
- 数据质量的评估结果

### 8.3 特征集说明

最终用于建模的特征列表，包括：
- 特征名称和描述
- 特征类型（数值/分类）
- 特征的来源（原始/衍生）
- 特征的预处理方式
