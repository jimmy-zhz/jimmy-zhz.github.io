# 探索性数据分析（EDA）阶段的数据处理：权威知识手册

---

## 1. EDA 的定义与理论框架

### 1.1 探索性数据分析的概念起源

**探索性数据分析（Exploratory Data Analysis，EDA）** 是由美国统计学家 **John Wilder Tukey**（1915-2000）在 1970 年代提出的统计思想。他在 1977 年出版的经典著作《*Exploratory Data Analysis*》中系统阐述了这一理念。

EDA 的核心思想是：**在正式建模之前，通过非正式、灵活的方式探索数据，发现模式、检验假设、生成新假设**。这与当时主流的"先假设后验证"的确认性数据分析（CDA）形成鲜明对比。

Tukey 认为，数据分析应该是一个**迭代的、探索性的过程**，而非线性的、预设的流程。分析师需要"与数据对话"，通过可视化、摘要统计、模式识别等手段，逐步理解数据的内在结构。

### 1.2 EDA 的哲学基础

EDA 建立在以下哲学原则之上：

| 原则 | 含义 | 实践意义 |
|------|------|---------|
| **数据优先** | 让数据自己说话，避免先入为主的假设 | 先探索后建模 |
| **灵活性** | 分析方法应随数据特性调整 | 不拘泥于固定流程 |
| **可视化思维** | 图形是发现隐藏模式的关键工具 | 重视图表而非仅看数字 |
| **怀疑精神** | 对数据质量保持质疑，警惕异常 | 系统性检查数据问题 |
| **迭代深化** | 分析是循环往复、逐步深入的过程 | 多次回顾和修正 |

### 1.3 EDA 在机器学习生命周期中的定位

在 CRISP-DM（跨行业数据挖掘标准流程）框架中，EDA 属于 **"数据理解"（Data Understanding）** 阶段的核心任务：

```
CRISP-DM 流程：
┌─────────────┐
│  业务理解   │  ← 理解项目目标
└──────┬──────┘
       ↓

│  (EDA)      │
└──────┬──────┘
       ↓
┌─────────────┐
│  数据准备   │  ← 基于 EDA 洞察进行清理

       ↓
```

### 1.4 EDA 的三大核心任务

根据现代数据科学实践，EDA 包含以下相互关联的任务：

#### 任务一：数据质量评估（Data Quality Assessment）

目标是识别和量化数据中的各类问题，包括：
- 缺失值（Missing Values）
- 异常值（Outliers）
- 重复记录（Duplicates）
- 矛盾数据（Inconsistent Records）
- 无效占位符（Invalid Placeholders）

#### 任务二：分布与统计特性分析（Distribution Analysis）

目标是理解数据的统计特性：
- 集中趋势（Central Tendency）
- 离散程度（Dispersion）
- 分布形态（Distribution Shape）
- 变量类型和取值范围

#### 任务三：变量关系探索（Relationship Exploration）

目标是揭示变量之间的关联：
- 相关性分析（Correlation Analysis）
- 条件依赖（Conditional Dependencies）
- 交互效应（Interaction Effects）
- 潜在结构（Latent Structure）

## 2. 数据质量评估

数据质量是机器学习模型性能的基石。"垃圾进，垃圾出"（Garbage In, Garbage Out）是数据科学的基本准则。高质量的数据应当满足以下特性：

| 特性 | 定义 | 评估方法 |
|------|------|---------|
| **完整性（Completeness）** | 数据的完整程度 | 缺失值比例 |
| **准确性（Accuracy）** | 数据与真实值的符合程度 | 与可靠来源对比 |
| **一致性（Consistency）** | 数据内部无矛盾 | 业务规则验证 |
| **时效性（Timeliness）** | 数据是否最新 | 时间戳分析 |
| **唯一性（Uniqueness）** | 无重复记录 | 重复检测 |
| **有效性（Validity）** | 数据符合定义的格式和范围 | 范围检查 |

### 2.1 缺失值（Missing Values）

#### 2.1.1 缺失值的定义与分类

缺失值是指在数据集中本应存在但实际缺失的数据。从统计学角度，缺失值可分为三种机制：

**（1）完全随机缺失（Missing Completely At Random, MCAR）**

缺失与任何变量（观测或未观测）都无关。

- **示例**：问卷在邮寄过程中随机丢失
- **影响**：删除缺失记录不会引入偏差，但会降低样本量

**（2）随机缺失（Missing At Random, MAR）**

缺失与观测变量相关，但不依赖于缺失变量本身的值。

- **示例**：高收入人群更倾向于不填写收入字段（但收入值本身不决定是否缺失）
- **影响**：简单删除会引入偏差，需要使用高级插补方法

**（3）非随机缺失（Missing Not At Random, MNAR）**

缺失与缺失变量本身的未观测值相关。

- **示例**：病情严重的患者更可能错过复诊（缺失与病情严重程度相关）
- **影响**：任何插补方法都可能产生偏差，需要敏感性分析

#### 2.1.2 缺失值的来源与识别

在实际工业数据中，缺失值可能以多种形式出现：

| 数据类型 | 常见占位符形式 | 典型来源 |
|---------|--------------|---------|
| **分类型** | `"?"`、`"Unknown"`、`"N/A"`、`""`、`" "`、`"Not Available"` | 用户调查、表单填写、遗留系统 |
| **数值型** | `-1`、`-99`、`-999`、`99`、`999`、`0`（逻辑上不可能为0时） | 传感器故障、默认值、数据导出 |
| **时间型** | `1970-01-01`（Unix纪元）、`0001-01-01`、`9999-12-31` | 日期转换失败、遗留数据库 |
| **浮点型** | `NaN`、`+inf`、`-inf`、`null` | IEEE 754 标准、计算溢出 |

**识别方法**：
- 物理缺失：`NaN`、`NULL` 等显式缺失标记
- 逻辑缺失：需要结合业务知识识别隐式占位符

#### 2.1.3 缺失值的有信息性

**重要概念**：缺失本身可能携带信息（Informative Missingness）

在某些情况下，缺失值与目标变量存在关联，缺失这一事实本身就具有预测价值：

- **客户数据**：收入字段缺失可能暗示客户对财务信息敏感
- **医疗数据**：某些检查项目缺失可能与病情严重程度相关
- **用户行为**：最后登录时间缺失可能表示用户已流失

**处理策略**：创建 **缺失指示变量（Missing Indicator）**，将"是否缺失"本身作为一个特征。

#### 2.1.4 缺失值的处理策略

| 策略 | 适用场景 | 潜在问题 |
|------|---------|---------|
| **删除记录** | MCAR、缺失比例低（<5%） | 可能损失有价值的信息 |
| **删除特征** | 缺失比例极高（>50%） | 可能丢失重要特征 |
| **均值/中位数填充** | MCAR、数值型 | 降低方差估计 |
| **众数填充** | 分类型 | 可能扭曲类别分布 |
| **插值法** | 时间序列数据 | 不适合随机缺失 |
| **KNN插补** | MAR | 计算成本高 |
| **多重插补（MICE）** | MAR | 方法复杂，需要多次建模 |
| **创建缺失标记** | MNAR或MAR | 增加特征维度 |

### 2.2 异常值（Outliers）

#### 2.2.1 异常值的统计学定义

异常值（Outlier）是指数据集中显著偏离其他观测值的个别记录。根据 Hawkins（1980）的经典定义：

> "异常值是指与其他观测值如此不一致，以至于让人怀疑它是由不同的机制产生的。"

异常值的成因通常包括：
- **数据错误**：录入错误、测量误差、传感器故障
- **真实极端值**：真实的极端现象（如超级富豪、极端天气）
- **混合分布**：数据来自多个不同总体

#### 2.2.2 异常值检测方法

**方法一：Z-Score 法**

基于正态分布假设，计算每个观测值与均值的标准差距离：

$$Z_i = \frac{x_i - \bar{x}}{s}$$

通常认为 $|Z| > 3$ 的观测值为异常值。

- **优点**：直观、易于理解
- **缺点**：假设正态分布；均值和标准差本身受异常值影响

**方法二：IQR（四分位距）法**

基于顺序统计量，不依赖分布假设：

$$\text{IQR} = Q_3 - Q_1$$

异常值定义为：
- $x < Q_1 - 1.5 \times \text{IQR}$
- $x > Q_3 + 1.5 \times \text{IQR}$

- **优点**：对异常值鲁棒；不假设分布
- **缺点**：只考虑边缘值，可能遗漏中等程度的异常

**方法三：修改 Z-Score 法**

使用中位数和绝对中位差（MAD）代替均值和标准差：

$$M_i = \frac{0.6745(x_i - \tilde{x})}{\text{MAD}}$$

其中 $\text{MAD} = \text{median}(|x_i - \tilde{x}|)$。

- **优点**：对非正态分布更加鲁棒
- **缺点**：计算稍复杂

**方法四：基于密度的检测（LOF）**

局部异常因子（Local Outlier Factor）通过比较每个点与其邻域的密度来检测异常。

- **优点**：能检测多维异常；发现局部异常
- **缺点**：需要参数调优；计算成本高

**方法五：Isolation Forest**

基于随机森林的异常检测方法，异常点更容易被"隔离"。

- **优点**：对高维数据有效；计算效率高
- **缺点**：解释性较弱

#### 2.2.3 异常值的处理原则

**核心原则**：统计异常 ≠ 数据错误

处理异常值时应遵循以下决策流程：

```
1. 验证异常值
   ├─ 是数据错误？→ 删除或修正
   └─ 不是数据错误？→ 下一步

2. 分析异常值的性质
   ├─ 真实极端值 → 保留，考虑变换或分箱
   └─ 混合分布 → 考虑分离建模

3. 考虑模型影响
   ├─ 树模型 → 可保留（对异常值鲁棒）
   └─ 线性模型 → 考虑截断、变换或删除
```

### 2.3 零方差与近似零方差特征

#### 2.3.1 定义

- **零方差特征（Zero-Variance）**：所有观测值都相同的特征，方差为 0
- **近似零方差特征（Near-Zero-Variance）**：某一类别占比超过 95% 的特征

#### 2.3.2 影响

这类特征对模型预测几乎没有贡献：
- 不提供任何区分信息
- 增加计算成本
- 可能导致数值问题（如在某些算法中除以零）

#### 2.3.3 处理策略

| 情况 | 建议 | 理由 |
|------|------|------|
| 零方差 | 删除 | 完全无信息量 |
| 近零方差（单一类别>95%） | 考虑删除 | 信息量极低 |
| 近零方差但与目标相关 | 保留 | 可能包含关键信号 |
| 树模型 | 可保留 | 对此类特征不敏感 |

### 2.4 重复记录

#### 2.4.1 类型

- **完全重复**：所有字段值都相同的记录
- **部分重复**：关键标识字段相同（如用户ID、交易ID），其他字段可能不同

#### 2.4.2 成因与影响

- **成因**：数据采集重复、ETL 流程问题、并发写入
- **影响**：权重偏差（重复记录被重复学习）、性能下降

#### 2.4.3 处理策略

- **删除重复**：保留一条记录（通常保留最新或最完整的一条）
- **聚合处理**：对重复记录进行汇总（如求和、取平均）
- **标记保留**：创建"重复标记"特征，保留信息

### 2.5 数据一致性与逻辑错误

#### 2.5.1 业务规则验证

数据应符合业务逻辑定义的规则：

| 规则类型 | 示例 |
|---------|------|
| **范围规则** | 年龄应在 0-120 岁之间 |
| **一致性规则** | 结束日期应晚于开始日期 |
| **唯一性规则** | 身份证号应唯一 |
| **参照完整性** | 订单中的商品ID应存在于商品表中 |
| **业务逻辑** | 折扣金额不应超过原价 |

#### 2.5.2 矛盾检测

- **字段间矛盾**：同一记录中字段值相互矛盾
- **时间矛盾**：时间顺序不符合逻辑
- **分布矛盾**：与已知总体分布显著偏离

## 3. 单变量统计分析

### 3.1 描述统计量

#### 3.1.1 集中趋势（Central Tendency）

| 统计量 | 定义 | 适用场景 | 对异常值的敏感性 |
|--------|------|---------|----------------|
| **均值（Mean）** | $\bar{x} = \frac{1}{n}\sum_{i=1}^{n} x_i$ | 对称分布 | 高度敏感 |
| **中位数（Median）** | 将数据分为两半的中间值 | 偏态分布、有异常值 | 鲁棒 |
| **众数（Mode）** | 出现频率最高的值 | 分类型数据 | 鲁棒 |
| **分位数（Quantile）** | 特定百分位的值 | 偏态分布 | 鲁棒 |

#### 3.1.2 离散程度（Dispersion）

| 统计量 | 定义 | 含义 |
|--------|------|------|
| **方差（Variance）** | $\sigma^2 = \frac{1}{n}\sum(x_i - \bar{x})^2$ | 数据的分散程度 |
| **标准差（Standard Deviation）** | $\sigma = \sqrt{\sigma^2}$ | 与原数据同单位 |
| **极差（Range）** | $max - min$ | 数据跨度 |
| **四分位距（IQR）** | $Q_3 - Q_1$ | 中间50%的跨度 |
| **变异系数（CV）** | $\frac{\sigma}{\bar{x}} \times 100\%$ | 无量纲离散度比较 |

#### 3.1.3 分布形态（Distribution Shape）

**偏度（Skewness）**：

$$\gamma_1 = \frac{1}{n}\sum(\frac{x_i - \bar{x}}{\sigma})^3$$

- $\gamma_1 > 0$：**右偏（正偏）**，长尾在右侧
- $\gamma_1 < 0$：**左偏（负偏）**，长尾在左侧
- $\gamma_1 \approx 0$：近似对称

**峰度（Kurtosis）**：

$$\gamma_2 = \frac{1}{n}\sum(\frac{x_i - \bar{x}}{\sigma})^4 - 3$$

- $\gamma_2 > 0$：**尖峰（Leptokurtic）**，重尾分布
- $\gamma_2 < 0$：**平峰（Platykurtic）**，轻尾分布
- $\gamma_2 = 0$：正态分布的峰度

### 3.2 数值变量的分布类型

| 分布类型 | 特征 | 常见场景 | 建模建议 |
|---------|------|---------|---------|
| **正态分布** | 对称、钟形 | 测量误差、身高 | 线性模型可直接使用 |
| **对数正态** | 右偏 | 收入、资产价格 | 对数变换后使用 |
| **指数分布** | 快速衰减 | 等待时间、故障时间 | 考虑专门模型 |
| **均匀分布** | 扁平 | 随机生成数据 | 考虑离散化 |
| **双峰/多峰** | 多个峰值 | 混合总体 | 考虑聚类或分段 |

### 3.3 分类变量的分析

#### 3.3.1 频率分布

- **频数（Frequency）**：每个类别出现的次数
- **频率（Proportion）**：频数除以总数
- **累积频率**：按类别排序的累积百分比

#### 3.3.2 类别数量问题

| 问题 | 阈值 | 处理建议 |
|------|------|---------|
| **高基数（High Cardinality）** | >50 个类别 | 特征分组、目标编码 |
| **类别不平衡** | 最大/最小 > 10:1 | 重采样、类权重 |
| **稀疏类别** | <1% | 合并为"其他"类 |

## 4. 多变量关系分析

### 4.1 相关性分析

#### 4.1.1 Pearson 相关系数

衡量两个连续变量之间的**线性关系**：

$$r = \frac{\sum(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum(x_i - \bar{x})^2}\sqrt{\sum(y_i - \bar{y})^2}}$$

- **取值范围**：$[-1, 1]$
- **解释**：
  - $|r| > 0.7$：强相关
  - $0.3 < |r| < 0.7$：中等相关
  - $|r| < 0.3$：弱相关

- **局限性**：仅捕捉线性关系；对异常值敏感

#### 4.1.2 Spearman 秩相关系数

衡量两个变量之间的**单调关系**（不一定是线性）：

- 基于数据的秩次（排名）计算
- 对异常值鲁棒
- 适用于序数数据和偏态分布

#### 4.1.3 Kendall Tau 相关系数

基于** concordant 和 discordant 对**的一致性度量：

- 对小样本更稳健
- 计算效率较低
- 适用于序数变量

#### 4.1.4 Cramér's V

衡量两个**分类变量**之间的关联强度：

$$V = \sqrt{\frac{\chi^2}{n \times \min(k-1, r-1)}}$$

其中 $\chi^2$ 是卡方统计量，$k$ 和 $r$ 是两个变量的类别数。

- **取值范围**：$[0, 1]$
- 解释标准：$V > 0.5$ 强关联，$0.3 < V < 0.5$ 中等关联

### 4.2 多重共线性（Multicollinearity）

#### 4.2.1 定义

多重共线性是指两个或多个自变量之间存在高度线性相关的情况。

#### 4.2.2 影响

- 线性模型中参数估计不稳定
- 方差膨胀，导致显著性检验失效
- 模型解释困难

#### 4.2.3 检测方法

| 方法 | 阈值 | 说明 |
|------|------|------|
| **相关系数** | > 0.9 | 简单快速 |
| **VIF（方差膨胀因子）** | > 5 或 10 | 经典方法 |
| **特征根检验** | 接近 0 | 严格标准 |

**VIF 计算**：

$$\text{VIF}_i = \frac{1}{1 - R_i^2}$$

其中 $R_i^2$ 是将第 $i$ 个变量对其他变量回归的 $R^2$。

#### 4.2.4 处理策略

- 删除高度相关的变量
- 使用正则化方法（Ridge、Lasso）
- 合并为综合指标（PCA）
- 保持相关（对树模型无影响）

### 4.3 变量与目标的关系

#### 4.3.1 分类目标

- **卡方检验**：检验类别变量与分类目标的独立性
- **ANOVA**：检验数值变量在不同目标类别间的均值差异
- **信息增益/互信息**：衡量特征对目标不确定性的减少程度

#### 4.3.2 连续目标

- **相关系数**：线性关系强度
- **回归系数**：控制其他变量后的影响
- **散点图可视化**：发现非线性关系

## 5. 数据清理策略

### 5.1 缺失值处理决策框架

```
缺失值分析流程：

Step 1: 评估缺失程度
  ├─ 缺失 > 50% → 删除该特征
  └─ 缺失 ≤ 50% → Step 2

Step 2: 判断缺失机制
  ├─ MCAR → 简单插补或删除
  ├─ MAR → 高级插补（KNN、MICE）
  └─ MNAR → 敏感性分析 + 领域知识

Step 3: 选择处理方法
  ├─ 数值型：均值/中位数/KNN/回归/MICE
  ├─ 分类型：众数/缺失标记/多重插补
  └─ 时间型：前向填充/插值

Step 4: 验证处理结果
  ├─ 分布是否改变
  └─ 变量关系是否保持
```

### 5.2 异常值处理策略

| 策略 | 方法 | 适用场景 |
|------|------|---------|
| **删除** | 直接移除异常记录 | 明确的数据错误 |
| **截断（Winsorization）** | 将超出阈值的值替换为阈值 | 保留样本量 |
| **变换** | 对数、Box-Cox 变换 | 重尾分布 |
| **分箱** | 将连续变量离散化 | 减少异常影响 |
| **标记** | 创建异常标记特征 | 异常值有信息价值 |
| **单独建模** | 为异常值单独建模 | 异常值来自不同总体 |

### 5.3 数据变换

#### 5.3.1 标准化（Standardization）

$$z = \frac{x - \mu}{\sigma}$$

- 均值变为 0，标准差变为 1
- 保留分布形态
- 适用于有量纲差异的特征

#### 5.3.2 归一化（Normalization）

$$x' = \frac{x - x_{min}}{x_{max} - x_{min}}$$

- 映射到 [0, 1] 区间
- 受极值影响大
- 适用于有明确边界的特征

#### 5.3.3 对数变换

$$x' = \log(x + c)$$

- 压缩右偏分布
- 稳定方差
- 适用于金融数据（收入、价格）

#### 5.3.4 Box-Cox 变换

$$x'(\lambda) = \begin{cases} \frac{x^\lambda - 1}{\lambda} & \lambda \neq 0 \\ \log(x) & \lambda = 0 \end{cases}$$

- 自动寻找最优变换参数
- 要求数据为正数
- 使数据更接近正态分布

## 6. 特征工程基础

### 6.1 特征类型

| 类型 | 定义 | 示例 |
|------|------|------|
| **数值特征** | 连续或离散的数值 | 年龄、收入、购买次数 |
| **分类特征** | 有限类别的值 | 性别、职业、国家 |
| **序数特征** | 有顺序的类别 | 学历（小学<中学<大学）、满意度 |
| **时间特征** | 与时间相关的特征 | 年、月、日、星期、年龄段 |
| **文本特征** | 自然语言文本 | 评论、标题、描述 |
| **图像特征** | 图像数据 | 照片、医学影像 |
| **地理特征** | 地理位置 | 经纬度、地址、城市 |

### 6.2 特征编码

#### 6.2.1 分类特征的编码

| 方法 | 描述 | 适用场景 |
|------|------|---------|
| **独热编码（One-Hot）** | 每个类别创建二元列 | 低基数、无序类别 |
| **标签编码（Label）** | 整数替换类别 | 高基数、树模型 |
| **目标编码（Target）** | 用目标均值替换 | 高基数、避免维度爆炸 |
| **有序编码（Ordinal）** | 利用类别顺序 | 序数特征 |
| **频率编码** | 用类别频率替换 | 捕捉类别稀有性 |

#### 6.2.2 数值特征的离散化

将连续变量转换为分类变量：

| 方法 | 描述 | 优点 |
|------|------|------|
| **等宽分箱** | 将范围等分 | 简单直观 |
| **等频分箱** | 每箱样本数相等 | 适应数据分布 |
| **决策树分箱** | 用决策树确定切分点 | 最优分割 |

### 6.3 特征选择

#### 6.3.1 过滤法（Filter Methods）

基于统计指标选择特征：

- **方差阈值**：删除低方差特征
- **相关性**：删除与目标低相关的特征
- **卡方检验**：分类特征选择
- **互信息**：捕捉非线性关系

#### 6.3.2 包装法（Wrapper Methods）

使用模型评估特征子集：

- **递归特征消除（RFE）**：逐步删除最不重要特征
- **前向选择**：逐步添加最有价值特征

#### 6.3.3 嵌入法（Embedded Methods）

在模型训练中自动选择特征：

- **Lasso 正则化**：L1 正则化产生稀疏解
- **树模型特征重要性**：基于分裂增益
- **特征选择集成**：如 SelectFromModel
