

## Experimental Design（实验设计）
机器学习的代码量虽然少，但看到的都是精简后的最优代码，背后是一个个被弃用的方案。

如果你想追求极致的模型，你会发现需要尝试的组合多得惊人：

- **特征处理**（不删 `duration` vs 删 `duration`）× 2
    
- **不平衡处理**（Class Weight vs SMOTENC vs 不处理）× 3
    
- **编码方式**（One-Hot vs Label Encoding）× 2
    
- **模型选择**（Random Forest vs XGBoost vs MLP）× 3
    
- **超参数微调**（每个模型又有无数参数）

#### 如何高效应对这种“体力活”？

为了不被无穷无尽的尝试淹没，我们会使用以下三个利器：

#### A. 从 Baseline（基准）出发，控制变量

不要一上来就尝试所有组合。先定一个最简单的“穷人版”流程（比如：删 `duration` + One-Hot + Class Weight + 默认参数随机森林）。 **以此为锚点，每次只改动一个变量。** 如果改了之后 F1-Score 没提升，直接放弃这个分支。

#### B. 自动化搜索 (Grid Search / Random Search)

不要手动去试参数！利用 `sklearn.model_selection.GridSearchCV`。 你只需要把想尝试的参数写成一个字典（Dictionary），程序就会自动帮你把这个“笛卡尔积”跑完，并告诉你哪组表现最好。

#### C. 使用 Pipeline（流水线）

这是你现在最需要的。把 `Scaler`、`Encoding`、`Weighting` 和 `Model` 全部打包进一个 `Pipeline` 对象。 这样你改一个参数，整个流程会全自动重新跑一遍，不会出现“忘了给测试集做 Scaling”这种低级错误。
