
_Multi-Agent Systems (MAS) Patterns_

### 1. 计划-执行-评估 循环 (The Plan-Execute-Evaluate Pattern) 🔄

这几乎是工业界公认的“三省六部”现代版。

- **规划器 (Planner)**：对应“中书省”。负责将复杂指令拆解为具体步骤。
    
- **执行器 (Executor)**：对应“六部”。由不同的专才 Agent 去跑代码、查数据库。
    
- **评估器 (Evaluator/Judge)**：对应“门下省”。这是目前工业界最看重的环节，通常由能力最强的模型（如 GPT-4o 或 Claude 3.5 Sonnet）担任，负责对执行结果进行质量打分。
    

### 2. 演讲者-批评家 模式 (Speaker-Critic Pattern) 🎭

这更接近你之前提到的“议会制”雏形，常用于代码生成和创意写作。

- **模式：** 一个 Agent 生成内容，另一个 Agent 专门负责找茬（指出 Bug、安全漏洞或逻辑矛盾）。
    
- **共识：** 工业界发现，**“左右互搏”**产生的代码质量，远高于单个 Agent 一次生成的质量。
    

### 3. 多层级分发模式 (Hierarchical Orchestration) 🏗️

在处理大型企业级项目（比如你熟悉的 **Java 高并发系统重构**）时，人们通常采用这种模式：

- **领班 (Manager Agent)**：它不干活，只负责监控状态和任务路由。
    
- **工兵 (Worker Agents)**：专注于极小的原子任务。
    
- **共识：** “层级越深，稳定性越高”。这和中国古代的官僚体系逻辑惊人地一致——通过增加管理层级来对冲底层 Agent 的不确定性。
