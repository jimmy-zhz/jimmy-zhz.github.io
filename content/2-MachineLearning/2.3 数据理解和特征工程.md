# 深度 EDA：三层分析框架

很多人误解 EDA 只是画直方图、箱线图、看均值方差——这只是**浅层 EDA**。真正完整的 EDA 包含三层递进的分析：

## 第一层：描述性 EDA（基础）

- 数据类型检查
- 分布分析
    
- 异常值检测
    
- 目标变量分布
    

👉 这是“看数据长什么样”

---

## 第二层：关系型 EDA（探索）

- 特征 vs Target 的关联分析
- 特征之间的相关性
    
- 非线性关系探索
- 共线性检测
    
- 信息增益与互信息
    
- 单变量区分能力评估

👉 这是“看数据是否有预测能力”

---

## 第三层：建模导向 EDA（高级）

- Tree-based importance（基于树的特征重要性）
- Permutation importance（置换重要性）
- SHAP 值分析
- 交互效应识别
- 分群可分性分析
    

👉 这是“为建模做结构判断”

---

# 特征工程核心概念

## 特征的本质

**特征（Feature）** 是从原始数据中提取的、用于表示样本属性的信息。在机器学习中，特征作为模型输入，帮助模型学习数据中的模式和规律。

### 特征质量原则

| 原则 | 含义 | 示例 |
|------|------|------|
| **相关性（Relevance）** | 特征与预测目标相关 | 房价预测中"面积"相关，"门牌号"不相关 |
| **非冗余性（Non-Redundancy）** | 特征之间不高度重复 | 避免同时使用"年薪"和"月薪×12" |
| **简洁性（Simplicity）** | 用尽量少的有效特征 | 奥卡姆剃刀原则 |
| **稳定性（Stability）** | 在不同样本上表现一致 | 避免季节性过强的特征 |
| **可解释性（Interpretability）** | 能解释清楚特征意义 | 便于业务理解 |
| **无泄漏（No Data Leakage）** | 不使用未来信息 | 不能用"购买后行为"预测"是否购买" |
| **计算效率（Efficiency）** | 特征计算复杂度可控 | 避免复杂实时计算 |
| **健壮性（Robustness）** | 对少量噪音不敏感 | 避免过度依赖精确值 |

### 多重共线性

**多重共线性（Multicollinearity）** 指特征之间存在高度线性相关：

- 例如：房间数与房屋面积高度相关
- **检测方法**：
  - 相关系数矩阵
  - 方差膨胀因子（VIF, Variance Inflation Factor）

# 特征提取（Feature Extraction）
特征提取是从原始数据中<font style="background-color:#FBDE28;color:black">提取出有用的信息</font>，以便机器学习模型可以使用这些信息进行训练和预测。
	
- 1️⃣ **图像特征提取**：
	+ **边缘检测**：例如使用Canny边缘检测算法，提取图像中的边缘特征。
	+ **SIFT、SURF、ORB**：用于提取局部特征点，广泛用于计算机视觉领域。
	+ **卷积神经网络（CNN）**：通过卷积层自动提取图像特征，是现代图像识别方法的基础。
	
- **2️⃣ 文本特征提取**：
	+ **TF-IDF（Term Frequency-Inverse Document Frequency）**：衡量一个词语在文本中的重要性，用于文本分类和信息检索。
	+ **Word2Vec、GloVe**：这些方法通过词嵌入将词语转换为向量，并捕捉词语之间的语义关系。
	+ **BERT、GPT**：基于深度学习的语言模型，能够提取上下文信息，捕捉更丰富的文本特征。
		
- **3️⃣ 时间序列特征提取**：
	+ **移动平均**：平滑时间序列数据，去除噪声。
	+ **傅里叶变换**：提取周期性信号的频域特征。
	+ **自回归模型（AR）、滑动平均模型（MA）**：用于提取时间序列中的依赖关系。
		
#### 特征选择方法论

特征选择是从所有可用特征中选取最重要特征的过程，主要有三大类方法：

| 方法 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| **过滤法（Filter）** | 基于统计指标（皮尔逊相关系数、卡方检验、互信息等） | 计算简单、速度快 | 忽略特征间相互作用 | 大规模数据集初筛 |
| **包裹法（Wrapper）** | 使用模型评估特征子集性能（如RFE） | 考虑特征间相互作用 | 计算开销大、易过拟合 | 中小规模数据集 |
| **嵌入法（Embedded）** | 将特征选择嵌入模型训练（Lasso、树模型重要性） | 效率与效果平衡 | 受模型限制 | 生产环境推荐 |

**常用算法**：
- **L1 正则化（Lasso）**：使部分特征系数为零，实现自动特征选择
- **随机森林特征重要性**：基于树分裂的信息增益评估
- **PCA**：虽主要用于降维，也可视为一种特征变换方法
	
#### 特征构造（Feature Construction）

特征构造是通过现有特征创建新特征的过程：

| 方法 | 描述 | 示例 |
|------|------|------|
| **数学运算** | 加减乘除运算 | 房价/面积 = 每平米价格 |
| **组合特征** | 多特征复合 | 收入/年龄 = 收入年龄比 |
| **时间衍生** | 从时间戳提取 | 从日期提取星期几、月份 |
| **分类转换** | 离散化或编码 | 年龄分段、One-Hot编码 |
| **聚合特征** | 类别统计 | 用户购买行为的均值、标准差 |
	
#### 特征变换（Feature Transformation）

特征变换是对数据特征进行数学处理，使其更适合模型训练：

##### 数值变换

| 方法 | 公式 | 目的 | 适用场景 |
|------|------|------|---------|
| **标准化** | $x' = \frac{x - \mu}{\sigma}$ | 转为均值0、标准差1 | 距离敏感算法（KNN、SVM、神经网络） |
| **归一化** | $x' = \frac{x - \min}{\max - \min}$ | 缩放到 [0,1] | 需要统一量纲的场景 |
| **对数变换** | $x' = \log(x+1)$ | 处理偏态分布 | 收入、价格等长尾数据 |
| **平方根变换** | $x' = \sqrt{x}$ | 温和分布矫正 | 轻微偏态数据 |
| **Box-Cox** | $y' = \frac{y^\lambda - 1}{\lambda}$ | 最优幂变换使数据正态化 | 线性回归等假设正态性的模型 |

##### 类别变换

| 方法 | 描述 | 适用场景 |
|------|------|---------|
| **One-Hot 编码** | 每个类别转为二进制向量 | 类别无序且数量较少 |
| **标签编码** | 类别映射为整数 | 类别有序或数量较多 |
| **目标编码** | 用目标变量均值编码 | 高基数类别特征 |

##### 降维变换

| 方法 | 描述 | 适用场景 |
|------|------|---------|
| **PCA** | 主成分分析，保留最大方差方向 | 线性降维、去除冗余 |
| **t-SNE/UMAP** | 非线性降维 | 高维数据可视化 |

# 数据增强（Data Augmentation）

## 定义与目的

数据增强是在不增加真实数据量的情况下，通过对原始数据进行变换来：
- 减少过拟合，提高模型泛化能力
- 增加训练数据的多样性
- 提高模型对噪声的鲁棒性

## 常见方法

| 数据类型 | 增强方法 |
|----------|---------|
| **图像** | 旋转、翻转、裁剪、颜色变换、MixUp、CutMix |
| **文本** | 同义词替换、回译、随机插入/删除 |
| **音频** | 变调、变速、添加噪声、时间拉伸 |
| **时间序列** | 窗口切片、噪声注入、时间扭曲 |

> **注意**：数据增强会消耗额外计算资源，可能导致训练时间增加。

# 表示学习理论

## 表示（Representation）

**表示**是将原始数据转化为计算机可处理的数值形式的过程：

- 文本 "猫" → 数字编码
- 图像 → 28×28 的向量矩阵

## 表征（Embedding）

**表征**是在表示基础上进一步提取有价值信息的过程：

- 猫的图像 → 深度学习提取出"动物属性 0.8"、"体型偏小 0.3"等语义特征
- 人脸图像 → 提取"眼睛间距"、"鼻梁角度"等隐含特征

在深度学习中，表征通常由模型**自动学习**得到。

## 局部表示 vs 分布式表示

### 局部表示（Local Representation）

又称离散表示或符号表示。典型例子是**独热编码（One-Hot）**：

- 猫 → `[1, 0, 0]`
- 狗 → `[0, 1, 0]`
- 鸟 → `[0, 0, 1]`

**特点**：
- 每个向量只有一位为 1，其余为 0
- 维度等于类别数量
- 可解释性强，但维度高且稀疏

### 分布式表示（Distributed Representation）

将数据表示为多维稠密向量，每个维度包含部分信息：

- 颜色 RGB 表示：`[255, 0, 0]` 表示红色
- 词向量 Word2Vec：单词表示为低维稠密向量

**特点**：
- 维度远小于类别数量
- 语义信息分布在多个维度
- 能捕捉语义相似性

> 神经网络中的 **Embedding Layer（嵌入层）** 是分布式表示的典型实现。

## 表征学习（Representation Learning）

自动化地从原始数据中学习有效特征表示（即**表征**）的方法。

深度学习的核心优势在于：**模型能够自动从数据中学习表征**，无需人工设计特征。

### 常见方法

1. **监督学习下的表征学习**
   - CNN 中间层输出作为图像表征
   - Transformer 隐藏状态作为文本表征

2. **无监督/自监督表征学习**
   - 无需标注数据，通过数据内在结构学习表征
   - **自编码器（Autoencoder）**：通过重构输入数据学习压缩表征
   - **对比学习（Contrastive Learning）**：如SimCLR、MoCo
   - **生成模型**：如GAN、VAE

3. **迁移学习与预训练模型**
   - 在大规模数据上预训练模型（如BERT、ResNet），其学到的表征可迁移到小样本任务中
