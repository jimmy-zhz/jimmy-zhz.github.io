## 正向传播


```python
from keras import Sequential, Input, layers  

model = Sequential([  
    Input(shape = (MAX_LENGTH, )),  
    layers.Embedding(VOCAB_SIZE, EMBEDDING_DIM),  
    layers.GlobalAveragePooling1D(),  
    layers.Dense(24, activation="relu"),  
    layers.Dense(1, activation="sigmoid")  
])
```
以这段简单的神经网络代码为例

### 神经网络的层

如上代码片段是一个简单的神经网络模型，总共有5层：
1. 输入层
2. 嵌入层
3. 池化层
4. 致密层（全连接层）
5. 输出层

但是从数学计算的角度，它只有3个"可学习层"

|**代码行**|**物理角色**|**数学/公式层号**|**理由**|
|---|---|---|---|
|`Embedding`|隐藏层 1|**Layer 1**|它有可学习的权重矩阵 $W^{(1)}$。|
|`GlobalAvgPool`|降维算子|(无独立层号)|它没有可学习参数，通常被视为 Layer 1 的一个后处理步骤。|
|`Dense(24)`|隐藏层 2|**Layer 2**|它有自己的 $W^{(2)}$ 和 $b^{(2)}$。|
|`Dense(1)`|输出层|**Layer 3**|这是最终产生预测的一层，有 $W^{(3)}$ 和 $b^{(3)}$。|
### 正向传播的过程

正向传播（Forward Propagation）是神经网络从输入到输出的计算过程。对于上面的3层网络：

| **步骤** | **代码层级**              | **数学符号**                      | **动作**                               |
| ------ | --------------------- | ----------------------------- | ------------------------------------ |
| 1      | `Input` + `Embedding` | $x \rightarrow a^{(0)}$       | 数据输入并转化为向量表示                         |
| 2      | `Dense(24, relu)`     | $z^{(1)} = W^{(1)}a^{(0)} + b^{(1)}$<br>$a^{(1)} = \text{ReLU}(z^{(1)})$ | **24个神经元**进行线性变换并经过 **ReLU** 激活      |
| 3      | `Dense(1, sigmoid)`   | $z^{(2)} = W^{(2)}a^{(1)} + b^{(2)}$<br>$a^{(2)} = \sigma(z^{(2)})$ | 输出层进行线性变换并经过 **Sigmoid** 激活          |
| 4      | **Loss 计算**            | $L = (y - a^{(2)})^2$         | **仅在这一步**，拿预测值 $a^{(2)}$ 和真实值 $y$ 计算损失 |

重要知识点：
- 每个iteration（迭代），会完成一次完整的正向传播和反向传播
- 正向传播的目的是：计算模型的预测值
- loss函数：
	- 目的： 衡量模型预测值与真实值的差距
	- 发生在模型在输出层得到结果之后
	- 为反向传播提供"误差信号"


## 反向传播

了解反向传播的目的和手段：

在计算得到loss函数值时，也就是得到了模型预测结果与实际值的偏差，为了让模型预测更准确，需要让这个偏差，也就是loss函数减少到接近于 **最低值**，也就是loss函数的全局最小值。

- 全局最小值并不总是0，有的loss函数的最低值可能为负，像MSE的理想值就是0，说明模型预测值与实际完全一致

以MSE作为Loss函数为例 （假设有三个数学层）：
$$L = (y - a^{(2)}(z^{(2)}(x) ))^2$$
- $y$：数据样本的真实标签值
- $\hat{y} = a^{(2)}( z^{(2)}(x) )$：模型预测值
	- $z^{(2)}(x)$：模型在输出层的线性变换，$z^{(2)} = W^{(2)}a^{(1)} + b^{(2)}$
	- 方程中的 $x$ 表示数据集的一个样本

### 理论理想与工程现实的差距

从理论角度看，理想的损失函数应基于**完整数据分布**（complete data distribution），包含所有已观测的训练数据、验证数据以及未来可能出现的未知数据。然而，真实的数据分布是**不可知的**（unknown），因此理论上的最优损失函数无法直接计算。

**工程实践中的折中方案**：

在实际模型训练中，我们只能在**有限训练数据集**上执行梯度下降优化。我们追求的"全局最小值"实际上是在训练集上定义的**经验风险最小值**（empirical risk minimizer），而非理论上的**期望风险最小值**（expected risk minimizer）。

- **经验风险**（Empirical Risk）：基于有限训练样本计算的损失函数近似
- **期望风险**（Expected Risk）：基于完整数据分布的理论损失函数

**核心局限**：由于只能访问有限数据，我们观察到的损失函数曲线仅是真实期望风险函数的**有限样本估计**（finite-sample estimate）。

后续讨论将聚焦于如何在训练数据的有限函数空间内寻找有效的优化解。

### 优化目标与挑战

**核心优化目标**：在每次迭代中计算参数**梯度更新量**，通过参数更新使损失函数值收敛到**局部最小值**（local minimum），并尽可能接近**全局最小值**（global minimum）。

**核心优化策略**：**梯度下降算法**（Gradient Descent Algorithm）

对于复杂机器学习任务（如自然语言处理、计算机视觉），损失函数的**全局最小值**通常无法直接解析求解。主要原因如下：

1. **非凸优化问题**（Non-convex Optimization）：
   - 神经网络损失函数通常是**高度非凸的**（highly non-convex），存在多个局部最小值
   - 无法保证找到的局部最小值就是全局最小值

2. **高维优化挑战**（High-dimensional Optimization Challenges）：
   - **函数复杂性**：神经网络是深度嵌套的复合函数，其梯度方程构成复杂的**非线性方程组**
   - **参数规模**：现代神经网络参数数量极大（从数万到数千亿），直接求解 $N$ 维非线性方程组的计算复杂度为 $O(N^3)$ 量级
   - **计算资源限制**：直接解析求解在计算时间和内存需求上均不可行

### 迭代优化策略

由于无法直接解析求解，我们采用**迭代优化方法**（iterative optimization approach）：

1. **逐步逼近**：通过多次迭代，逐步调整参数使损失函数值下降
2. **梯度引导**：每次迭代沿负梯度方向更新参数：$\theta_{t+1} = \theta_t - \eta \nabla L(\theta_t)$
3. **收敛判断**：当损失函数值不再显著下降或达到预设迭代次数时停止

**优化过程类比**：

将损失函数空间想象为**多峰地形**（multi-modal landscape），每个局部最小值对应一个"山谷"。我们无法直接判断某个梯度为零的点是局部最小值还是全局最小值，但通过**多次探索**（multiple training epochs）和**不同初始化**，可以增加找到更优解的概率。

**关键洞察**：虽然无法保证找到理论全局最小值，但通过充分的迭代训练，通常能找到**足够好的实用解**（sufficiently good practical solution）。

### 梯度

梯度（Gradient）是一个向量，向量里的每个元素是损失函数 $L$ 对某个模型参数（$w$ 或 $b$）的偏导数。对于有 $n$ 个参数的模型，梯度向量为：

$$\nabla L = \left[ \frac{\partial L}{\partial w_1}, \frac{\partial L}{\partial w_2}, \dots, \frac{\partial L}{\partial w_n}, \frac{\partial L}{\partial b_1}, \frac{\partial L}{\partial b_2}, \dots \right]$$

注意：==**梯度是指所有模型参数在这个iteration内 对loss函数的偏导数的集合向量**==

在进行数学更新时，优化器会逻辑上把这些矩阵和向量摊平（Flatten），拼接成一个长串的 $\nabla L$，这就是梯度。

梯度的方向指向损失函数增长最快的方向。由于我们的目标是让损失函数减少，我们需要沿着梯度的反方向更新参数。

### 梯度下降

梯度下降（Gradient Descent）是优化模型参数的核心算法：

$$W_{new} = W_{old} - \eta \cdot \nabla L(W_{old})$$

其中：
- **$\eta$ (学习率, Learning Rate)**：控制更新步长的大小。这是在 PyTorch 优化器（如 `SGD(lr=0.01)`）里设置的超参数。
- **$\nabla L$ (梯度)**：损失函数在当前参数处的梯度向量，包含方向和陡峭程度信息。
- **乘积**："学习率 $\times$ 梯度" = **参数更新量**

某个参数 $w_i$ 在反向传播后需要更新多少，就是计算 $\frac{\partial L}{\partial w_i} \times \eta$。


1. **梯度的局部性与统计偏差**

梯度下降理论保证模型能收敛到局部最小值，但在实际训练中，每个迭代（iteration）仅基于一个小批量（mini-batch）数据计算梯度。这种基于有限样本的梯度估计存在**统计偏差**（statistical bias），仅反映了损失函数在当前数据子集上的局部特性，而非全局真实的梯度方向。

因此，每个迭代计算的梯度可视为一个**带有噪声的梯度估计**（noisy gradient estimate）。虽然其方向大致正确，但存在随机波动，不能完全信赖为精确的下降方向。

2. **学习率 ($\eta$) 的调节作用**

由于梯度估计存在噪声，不能完全信赖单次迭代的梯度方向。学习率的核心作用是**调节更新步长**，平衡梯度噪声的影响。

- **高学习率的风险**：若学习率过大，模型更新步长过长，可能因单次迭代的梯度噪声而**偏离优化路径**，甚至导致损失函数值上升（发散）。在极端情况下，可能完全错过局部最小值区域。

- **低学习率的优势**：较小的学习率通过**多次迭代的统计平均效应**（statistical averaging effect）来抵消梯度噪声。虽然单次更新方向可能存在偏差，但多次迭代的平均更新方向更接近真实的梯度下降方向，使优化过程更加稳定。

3. **学习率的动态平衡：优化效率与稳定性**

学习率的选择需要在**收敛速度**（convergence speed）和**优化稳定性**（optimization stability）之间取得平衡：

| **学习率设置** | **优化行为** | **训练表现** |
|--------------|------------|------------|
| **学习率过高** | 更新步长过大，可能在最小值附近**振荡**（oscillation）或**发散**（divergence） | 损失函数值剧烈波动，无法收敛，曲线呈锯齿状或发散趋势 |
| **学习率过低** | 更新步长过小，**收敛速度极慢**，需要大量迭代才能接近最小值 | 损失函数下降缓慢，训练时间过长，计算资源利用率低 |
| **学习率适中** | 稳定收敛到**宽谷区域**（wide basin）的局部最小值 | 损失函数平滑下降，呈现典型的指数衰减或L型收敛曲线 |

理想的学习率应使模型能够**稳定、高效地收敛**到损失函数的局部最小值区域。

### 链式法则

链式法则（Chain Rule）是反向传播的核心数学工具。由于神经网络是一个复合函数（多个函数嵌套），我们需要计算损失函数 $L$ 对每一层参数 $W^{(l)}$ 的偏导数。

![[chains_rule.jpg]]
如图展示，链式法则计算梯度向量中其中一个元素， 也就是$w_1$这个模型参数的偏导数， 的计算过程

对于第 $l$ 层的权重 $W^{(l)}$，其梯度计算如下：

$$\frac{\partial L}{\partial W^{(l)}} = \frac{\partial L}{\partial a^{(L)}} \times \frac{\partial a^{(L)}}{\partial z^{(L)}} \times \frac{\partial z^{(L)}}{\partial a^{(L-1)}} \times \cdots \times \frac{\partial a^{(l+1)}}{\partial z^{(l+1)}} \times \frac{\partial z^{(l+1)}}{\partial a^{(l)}} \times \frac{\partial a^{(l)}}{\partial z^{(l)}} \times \frac{\partial z^{(l)}}{\partial W^{(l)}}$$

其中：
- $(L)$ 是总层数
- $a^{(l)}$ 是第 $l$ 层的激活值
- $z^{(l)} = W^{(l)}a^{(l-1)} + b^{(l)}$ 是第 $l$ 层的线性变换

**链式法则在反向传播中的作用：**
1. **误差反向传播**：将输出层的误差逐层反向传播到每一层
2. **局部计算**：每一层只需要计算自己的局部梯度，然后与上游梯度相乘
3. **计算效率**：避免了重复计算，通过缓存中间结果提高效率

**实际计算过程（反向传播算法）：**
1. **前向传播**：计算每一层的 $z^{(l)}$ 和 $a^{(l)}$，并缓存这些值
2. **计算输出层梯度**：$\delta^{(L)} = \frac{\partial L}{\partial a^{(L)}} \cdot \sigma'(z^{(L)})$
3. **反向传播误差**：对于 $l = L-1, L-2, \dots, 1$：
   $$\delta^{(l)} = (W^{(l+1)})^T \delta^{(l+1)} \cdot \sigma'(z^{(l)})$$
4. **计算参数梯度**：
   $$\frac{\partial L}{\partial W^{(l)}} = \delta^{(l)} (a^{(l-1)})^T$$
   $$\frac{\partial L}{\partial b^{(l)}} = \delta^{(l)}$$

这种计算方式使得反向传播非常高效，是现代深度学习能够训练大规模神经网络的关键。

### 如何找到全局最小值 {#如何找到全局最小值}

没有一种算法能百分之百保证跳出所有陷阱（局部最小值），所以我们用统计学上的概率和工程上的技巧来对抗它

#### 1. 备份与多跑

- Epochs = 100 数据集多跑几次
- 留存历史最佳
```python
if(loss < history_min):
	model.save()
```
#### 2. **动量 (Momentum)**

现在的更新量不只看当前的梯度，还要加上一部分上次更新的“惯性动能”，使得在局部小值处，还能继续更新。


####  3. 随机性 (Stochasticity)

分批投喂，如32个样本一批。这 32 条数据形成的“局部地貌”可能会把那个小坑填平，或者把模型往旁边“推”一把。这种**噪声**反而成了模型跳出陷阱的动力。

#### 4. 自适应学习率 (Adam)

发现某个参数最近一段时间都没怎么动，Adam 会自动**放大**它的更新步长，尝试把它拉出泥潭。

#### 5. 跳跃连接 (Skip Connections)

这是 ResNet 等现代深度模型成功的核心。

- **物理手段**：如果在 5 层网络中，信号在中间层传丢了，我们就修一条“高速公路”让输入直接跳到后面。
    
- **客观影响**：这从根本上**改变了山脉的地形**，把原本崎岖不平、布满陷阱的 Loss 曲线，变得更加平缓、顺滑，从而减少了陷阱的数量。


### Backpropagation 逻辑闭环：

1. **裁判 (Loss)**：定义错误的大小。
    
2. **翻译 (Chain Rule)**：将总错误拆解，分配给网络中每一个 $w$ 和 $b$。
    
3. **导航 (Gradient)**：汇聚所有拆解后的错误，形成指向“更差方向”的向量。
    
4. **行动 (Optimizer)**：朝着导航的反方向（$-\eta$）迈出一小步，更新模型。
    
5. **纠偏 (Momentum/Adam)**：通过惯性和自适应，防止在小坑里停步。

## 实际代码示例：PyTorch中的正向传播与反向传播

下面是一个完整的PyTorch示例，展示正向传播和反向传播的实际实现：

```python
import torch
import torch.nn as nn
import torch.optim as optim

# 1. 定义简单的神经网络
class SimpleNN(nn.Module):
    def __init__(self, input_size=10, hidden_size=24, output_size=1):
        super(SimpleNN, self).__init__()
        self.fc1 = nn.Linear(input_size, hidden_size)  # Layer 1: W^{(1)}, b^{(1)}
        self.relu = nn.ReLU()                          # 激活函数
        self.fc2 = nn.Linear(hidden_size, output_size) # Layer 2: W^{(2)}, b^{(2)}
        self.sigmoid = nn.Sigmoid()                    # 输出激活函数
        
    def forward(self, x):
        # 正向传播过程
        z1 = self.fc1(x)      # z^{(1)} = W^{(1)}x + b^{(1)}
        a1 = self.relu(z1)    # a^{(1)} = ReLU(z^{(1)})
        z2 = self.fc2(a1)     # z^{(2)} = W^{(2)}a^{(1)} + b^{(2)}
        output = self.sigmoid(z2)  # a^{(2)} = σ(z^{(2)})
        return output

# 2. 创建模型、损失函数和优化器
model = SimpleNN()
criterion = nn.MSELoss()      # 损失函数: L = (y - ŷ)²
optimizer = optim.SGD(model.parameters(), lr=0.01)  # 优化器: W_new = W_old - η·∇L

# 3. 模拟训练数据
batch_size = 32
x = torch.randn(batch_size, 10)  # 输入数据: 32个样本，每个10维
y = torch.randn(batch_size, 1)   # 真实标签

# 4. 一个完整的训练迭代
print("=== 开始一个训练迭代 ===")

# 步骤1: 梯度清零（重要！）
optimizer.zero_grad()
print("1. 梯度已清零")

# 步骤2: 正向传播
output = model(x)
print(f"2. 正向传播完成，预测值形状: {output.shape}")

# 步骤3: 计算损失
loss = criterion(output, y)
print(f"3. 损失计算完成，Loss = {loss.item():.4f}")

# 步骤4: 反向传播（自动计算梯度）
loss.backward()
print("4. 反向传播完成，梯度已计算")

# 检查梯度
print("\n梯度检查:")
for name, param in model.named_parameters():
    if param.grad is not None:
        print(f"  {name}: 梯度范数 = {param.grad.norm().item():.6f}")

# 步骤5: 参数更新（梯度下降）
optimizer.step()
print("\n5. 参数更新完成")

# 步骤6: 验证参数已更新
print("\n参数更新验证:")
for name, param in model.named_parameters():
    print(f"  {name}: 参数已更新")
```

### 关键点解释：

1. **`zero_grad()`**：在每次反向传播前必须调用，否则梯度会累积
2. **`forward()`**：定义正向传播的计算图
3. **`loss.backward()`**：自动执行反向传播，计算所有参数的梯度
4. **`optimizer.step()`**：根据梯度更新参数：$W_{new} = W_{old} - \eta \cdot \nabla L$

### 手动计算梯度（理解原理）：

```python
# 手动验证梯度计算（仅用于理解）
def manual_backprop():
    # 假设我们有一个简单的线性模型: y = w*x + b
    w = torch.tensor([1.0], requires_grad=True)
    b = torch.tensor([0.5], requires_grad=True)
    x = torch.tensor([2.0])
    y_true = torch.tensor([3.0])
    
    # 正向传播
    y_pred = w * x + b  # y_pred = 1.0*2.0 + 0.5 = 2.5
    
    # 计算损失 (MSE)
    loss = (y_pred - y_true) ** 2  # (2.5 - 3.0)² = 0.25
    
    # 反向传播
    loss.backward()
    
    print(f"手动计算梯度:")
    print(f"  ∂L/∂w = {w.grad.item():.2f}")  # 2*(y_pred-y_true)*x = 2*(2.5-3.0)*2.0 = -2.0
    print(f"  ∂L/∂b = {b.grad.item():.2f}")  # 2*(y_pred-y_true) = 2*(2.5-3.0) = -1.0
    
    # 参数更新 (lr=0.1)
    lr = 0.1
    w_new = w - lr * w.grad  # 1.0 - 0.1*(-2.0) = 1.2
    b_new = b - lr * b.grad  # 0.5 - 0.1*(-1.0) = 0.6
    
    print(f"更新后参数:")
    print(f"  w_new = {w_new.item():.2f}")
    print(f"  b_new = {b_new.item():.2f}")

manual_backprop()
```

这个示例展示了从理论到实践的完整过程，帮助你更好地理解正向传播和反向传播的实际实现。

## 总结

### 核心要点回顾

1. **正向传播**：
   - 目的：计算模型预测值
   - 过程：输入 → 线性变换 → 激活函数 → 输出
   - 数学：$z^{(l)} = W^{(l)}a^{(l-1)} + b^{(l)}$, $a^{(l)} = \sigma(z^{(l)})$

2. **反向传播**：
   - 目的：计算参数梯度，更新模型
   - 核心：链式法则 $\frac{\partial L}{\partial W^{(l)}} = \frac{\partial L}{\partial a^{(L)}} \times \cdots \times \frac{\partial z^{(l)}}{\partial W^{(l)}}$
   - 算法：误差反向传播，逐层计算梯度

3. **梯度下降**：
   - 公式：$W_{new} = W_{old} - \eta \cdot \nabla L$
   - 关键：学习率 $\eta$ 控制更新步长

4. **优化技巧**：
   - 动量：增加更新惯性，避免局部最小值
   - 随机性：小批量训练增加噪声，帮助跳出局部最小值
   - 自适应学习率：根据参数历史调整步长
   - 跳跃连接：改善梯度流动，缓解梯度消失

### 实践建议

1. **调试技巧**：
   - 使用 `torch.autograd.gradcheck()` 验证梯度计算
   - 监控梯度范数，避免梯度爆炸/消失
   - 可视化损失曲线，观察训练过程

2. **性能优化**：
   - 使用 GPU 加速计算
   - 合理设置批量大小
   - 选择合适的优化器（SGD, Adam, RMSprop）

3. **常见问题**：
   - **梯度消失**：使用 ReLU、BatchNorm、残差连接
   - **过拟合**：使用 Dropout、正则化、早停
   - **训练不稳定**：调整学习率、梯度裁剪

通过深入理解正向传播和反向传播的原理，你可以更好地设计、调试和优化神经网络模型。
