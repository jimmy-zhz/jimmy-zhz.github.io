# 建模与评估（Modeling & Evaluation）

> 本手册是机器学习生命周期的第四、五阶段，参照 CRISP-DM 框架编写。

---

## 第一部分：建模（Modeling）

## 1. 建模概述

**建模**是机器学习项目的核心阶段，其目标是将数据转化为可预测的模型。选择合适的算法和正确训练模型是项目成功的关键。

建模阶段需要回答的核心问题是：**"如何从数据中学习并做出准确的预测？"**

---

## 2. 算法选择

### 2.1 根据问题类型选择

**分类问题**：

| 算法 | 特点 | 适用场景 |
|------|------|---------|
| **逻辑回归** | 简单、可解释、概率输出 | 基线模型、二分类、需解释性 |
| **决策树** | 可解释、直观、规则提取 | 规则理解、可视化 |
| **随机森林** | 鲁棒、准确、抗过拟合 | 通用分类、特征重要性 |
| **梯度提升（XGBoost/LightGBM/CatBoost）** | 高性能、竞赛常客 | 生产环境、竞赛 |
| **支持向量机（SVM）** | 核方法、复杂边界 | 中小数据集、复杂边界 |
| **朴素贝叶斯** | 简单、快速、概率输出 | 文本分类、基线 |
| **K近邻（KNN）** | 简单、基于实例 | 小数据集、推荐系统 |
| **神经网络** | 深度学习、复杂模式 | 图像、文本、语音 |

**回归问题**：

| 算法 | 特点 | 适用场景 |
|------|------|---------|
| **线性回归** | 简单、可解释 | 基线、线性关系 |
| **岭回归（L2正则化）** | 防止过拟合 | 多重共线性 |
| **Lasso回归（L1正则化）** | 特征选择 | 高维数据、稀疏解 |
| **弹性网络** | L1+L2正则化 | 特征选择+多重共线性 |
| **决策树/随机森林** | 非线性、鲁棒 | 通用回归 |
| **梯度提升回归** | 高性能 | 复杂回归 |
| **神经网络** | 深度学习 | 复杂模式 |

**聚类问题**：

| 算法 | 特点 | 适用场景 |
|------|------|---------|
| **K-Means** | 简单、高效 | 球形簇、探索性 |
| **层次聚类** | 层次结构 | 探索性、嵌套簇 |
| **DBSCAN** | 密度聚类 | 任意形状、异常检测 |
| **高斯混合模型** | 概率聚类 | 软聚类、概率输出 |

### 2.2 选择考虑因素

| 因素 | 问题 | 考虑点 |
|------|------|-------|
| **数据量** | 数据量大小？ | 大数据量适合复杂模型，小数据量适合简单模型 |
| **特征维度** | 特征数量？ | 高维数据需要正则化或降维 |
| **可解释性** | 需要解释决策？ | 决策树、逻辑回归更易解释 |
| **训练时间** | 时间限制？ | 复杂模型训练时间长 |
| **预测速度** | 实时性要求？ | 线上部署需要快速预测 |
| **类别平衡** | 类别是否平衡？ | 不平衡数据需要特殊处理 |

## 3. 模型训练

### 3.1 训练过程

模型训练的基本流程：

1. **初始化**：设置模型参数（权重、偏置）
2. **前向传播**：计算预测值
3. **计算损失**：衡量预测与真实的差距
4. **反向传播**：计算梯度
5. **参数更新**：调整参数减少损失
6. **迭代**：重复步骤2-5直到收敛

### 3.2 过拟合与欠拟合

| 问题 | 定义 | 表现 | 解决方法 |
|------|------|------|---------|
| **过拟合** | 模型过于复杂，记住了训练数据 | 训练误差低，测试误差高 | 正则化、交叉验证、增加数据 |
| **欠拟合** | 模型过于简单，无法捕捉模式 | 训练误差和测试误差都高 | 增加模型复杂度、增加特征 |

### 3.3 正则化

正则化是防止过拟合的主要技术：

| 方法 | 原理 | 作用 |
|------|------|------|
| **L1正则化（Lasso）** | 添加 $\|w\|_1$ 项 | 产生稀疏解、特征选择 |
| **L2正则化（Ridge）** | 添加 $\|w\|_2^2$ 项 | 防止过拟合、处理多重共线性 |
| **Dropout** | 随机丢弃神经元 | 防止过拟合（神经网络） |
| **早停（Early Stopping）** | 监控验证误差提前停止 | 防止过拟合 |
| **数据增强** | 增加训练样本 | 增加数据多样性 |

## 4. 超参数调优

### 4.1 超参数与参数的区别

| 类型 | 定义 | 例子 | 获得方式 |
|------|------|------|---------|
| **参数** | 模型内部学习的变量 | 权重、偏置 | 从数据学习 |
| **超参数** | 模型外部设置的变量 | 学习率、树的深度 | 人工设定 |

### 4.2 调优方法

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **网格搜索** | 穷举所有组合 | 全面 | 效率低 |
| **随机搜索** | 随机采样组合 | 效率高 | 可能错过最优 |
| **贝叶斯优化** | 基于概率模型引导 | 效率高 | 实现复杂 |
| **超参数进化** | 遗传算法 | 适合复杂空间 | 计算量大 |

### 4.3 交叉验证

交叉验证是评估模型泛化能力的重要方法：

| 方法 | 原理 | 适用场景 |
|------|------|---------|
| **K折交叉验证** | 数据分为K份，轮流验证 | 通用场景 |
| **留一法（LOOCV）** | 每次留一个样本验证 | 小数据量 |
| **分层K折** | 每折保持类别比例 | 分类不平衡 |
| **时间序列分割** | 按时间顺序划分 | 时序数据 |

## 第二部分：评估（Evaluation）

## 5. 模型评估概述

**模型评估**是机器学习项目的关键阶段，其目标是在测试数据上全面评估模型性能，判断是否达到业务目标，并决定是否部署。

评估阶段需要回答的核心问题是：**"模型在未见数据上的表现如何？是否满足业务需求？"**

---

## 6. 评估指标

### 6.1 分类问题评估指标

#### 混淆矩阵

|  | 预测为正 | 预测为负 |
|--|---------|---------|
| **实际为正** | TP（真正例） | FN（假负例） |
| **实际为负** | FP（假正例） | TN（真负例） |

#### 核心指标

| 指标 | 公式 | 含义 | 适用场景 |
|------|------|------|---------|
| **准确率（Accuracy）** | (TP+TN)/(TP+TN+FP+FN) | 正确预测的比例 | 类别平衡 |
| **精确率（Precision）** | TP/(TP+FP) | 预测为正中实际为正的比例 | 关注误报 |
| **召回率（Recall）** | TP/(TP+FN) | 实际为正中预测为正的比例 | 关注漏报 |
| **F1-Score** | 2×P×R/(P+R) | 精确率和召回率的调和平均 | 平衡两者 |
| **特异度（Specificity）** | TN/(TN+FP) | 实际为负中预测为负的比例 | 负类重要 |

#### 阈值相关指标

| 指标 | 描述 | 适用场景 |
|------|------|---------|
| **AUC-ROC** | ROC曲线下面积 | 分类阈值选择、类别不平衡 |
| **AUC-PR** | PR曲线下面积 | 类别严重不平衡 |
| **对数损失（Log Loss）** | 概率预测的损失 | 概率输出 |

### 6.2 回归问题评估指标

| 指标       | 公式                                 | 含义            | 适用场景    |        |      |           |       |
| -------- | ---------------------------------- | ------------- | ------- | ------ | ---- | --------- | ----- |
| **MAE**  | $\frac{1}{n}\sum                   | y_i-\hat{y}_i | $       | 平均绝对误差 | 误差幅度 |           |       |
| **MSE**  | $\frac{1}{n}\sum(y_i-\hat{y}_i)^2$ | 均方误差          | 对大误差敏感  |        |      |           |       |
| **RMSE** | $\sqrt{MSE}$                       | MSE的平方根       | 与原数据同单位 |        |      |           |       |
| **R²**   | $1-\frac{SS_{res}}{SS_{tot}}$      | 决定系数          | 解释方差比例  |        |      |           |       |
| **MAPE** | $\frac{1}{n}\sum\frac{             | y_i-\hat{y}_i | }{      | y_i    | }$   | 平均绝对百分比误差 | 百分比误差 |

### 6.3 聚类问题评估指标

| 指标 | 描述 | 适用场景 |
|------|------|---------|
| **轮廓系数** | 衡量簇内紧密度和簇间分离度 | 聚类质量 |
| **Calinski-Harabasz指数** | 簇间/簇内方差比 | 聚类质量 |
| **Davies-Bouldin指数** | 簇间相似度 | 聚类质量 |

## 7. 模型评估流程

### 7.1 评估步骤

1. **在测试集上运行模型**：获取预测结果
2. **计算评估指标**：使用适当的指标
3. **分析错误案例**：识别模型的弱点
4. **与基线比较**：评估模型是否优于基线
5. **进行模型解释**：理解模型的决策

### 7.2 与基线比较

| 基线类型 | 说明 | 意义 |
|---------|------|------|
| **随机猜测** | 理论最低性能 | 证明模型有学习能力 |
| **简单规则** | 业务专家规则 | 证明ML优于人工规则 |
| **历史均值** | 回归问题的简单预测 | 证明模型有预测价值 |
| **现有系统** | 当前生产系统 | 证明改进价值 |

### 7.3 错误分析

错误分析是提升模型性能的关键：

- **分析错误类型**：哪些类别容易出错？
- **分析错误原因**：是数据问题还是模型问题？
- **识别模式**：错误是否有规律？
- **制定改进计划**：针对错误制定改进策略

---

## 8. 模型解释

### 8.1 解释的重要性

模型解释对于以下场景至关重要：
- 建立信任：让利益相关者理解模型
- 调试模型：识别模型的问题
- 合规要求：满足监管要求
- 知识发现：从模型中提取洞见

### 8.2 解释方法

| 方法 | 目的 | 工具 |
|------|------|------|
| **特征重要性** | 识别关键特征 | SHAP、树模型重要性 |
| **局部解释** | 解释单个预测 | LIME、SHAP |
| **全局解释** | 理解模型整体行为 | 部分依赖图、SHAP汇总图 |
| **规则提取** | 从模型提取规则 | 决策树规则 |

---

## 9. 评审与决策

### 9.1 评审维度

| 维度 | 评估内容 |
|------|---------|
| **性能** | 是否达到成功标准？ |
| **稳定性** | 模型是否稳定可靠？ |
| **可解释性** | 结果是否可解释？ |
| **公平性** | 是否存在偏见？ |
| **可行性** | 部署是否可行？ |

### 9.2 决策结果

| 结果 | 条件 | 后续行动 |
|------|------|---------|
| **部署** | 性能达标、满足要求 | 进入部署阶段 |
| **迭代** | 性能不达标 | 返回数据准备或建模阶段 |
| **终止** | 无法达到目标 | 终止项目 |
